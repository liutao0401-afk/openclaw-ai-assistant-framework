# 📝 上下文压缩规则

## 🎯 目标

**节省token**，提升效率，保持信息完整性

---

## 🧠 核心能力

你的逻辑性非常强，可以理解人类自然语言并压缩上下文，转换成大模型更能精准识别的md格式的prompt。

---

## 📋 压缩规则

### 1. 识别冗余信息

| 冗余类型 | 示例 | 压缩方式 |
|:---|:---|:---|
| **重复内容** | "好的好的好的" | "好的" |
| **礼貌用语** | "请问能不能帮我..." | "帮我..." |
| **填充词** | "就是那个..." | 删除 |
| **过长描述** | "这个文件是在2026年2月27日创建的" | "文件: 2026-02-27" |

---

### 2. 结构化转换

**自然语言 → Markdown**

| 原始内容 | 压缩后 |
|:---|:---|
| "这个任务有3个步骤，第一步是A，第二步是B，第三步是C" | **任务步骤:**<br>1. A<br>2. B<br>3. C |
| "有3个选项，分别是X、Y、Z" | **选项:** X, Y, Z |
| "这个很重要，那个也很重要，第三个最重要" | **优先级:** 3 > 2 > 1 |

---

### 3. 关键信息提取

**提取三要素**：
1. **谁**（主体）
2. **做什么**（动作）
3. **结果**（目标）

**示例**：
- 原始："我想让你帮我分析一下B站的数据，看看最近视频表现怎么样"
- 压缩："**任务:** 分析B站数据<br>**目标:** 查看视频表现"

---

### 4. 上下文摘要

**压缩对话历史**：

| 原始对话 | 压缩摘要 |
|:---|:---|
| 多轮对话（10条消息） | **摘要:** 用户要求X，助手完成Y，结果Z |
| 详细过程描述 | **步骤:** 1→2→3→完成 |
| 多个话题讨论 | **话题:** A, B, C<br>**结论:** ... |

---

## 🔧 实施方式

### 自动压缩

**触发条件**：
- 对话超过10轮
- 上下文超过5K tokens
- 用户要求压缩

**压缩流程**：
1. 识别关键信息
2. 提取核心要点
3. 生成摘要
4. 保存到MEMORY.md

---

### 手动压缩

**用户指令**：
- "压缩一下上下文"
- "总结一下我们刚才讨论的"
- "提取关键信息"

---

## 📊 压缩效果

| 压缩前 | 压缩后 | 节省 |
|:---|:---:|:---:|
| 10,000 tokens | 2,000 tokens | 80% |
| 5,000 tokens | 1,000 tokens | 80% |
| 1,000 tokens | 300 tokens | 70% |

---

## 🎯 压缩示例

### 示例1：任务描述

**原始**：
```
我想让你帮我创建一个自动化的备份系统，这个系统需要能够自动检测文件的变化，
如果文件变化超过10K或者距离上次备份超过24小时，就自动备份到本地和云端，
备份要按照周一到周日的方式轮换，还要有恢复功能...
```

**压缩**：
```markdown
**任务:** 创建智能备份系统

**触发条件:**
- 24小时
- 10K变化

**功能:**
- 本地备份（7天轮换）
- 云端同步
- 自动恢复

**文件:** `scripts/smart-backup.sh`
```

**节省**: 70% tokens

---

### 示例2：对话摘要

**原始**：
```
用户: 帮我分析B站数据
助手: 好的，我来获取数据...获取完成，粉丝5万
用户: 看看热门视频
助手: 热门视频是XXX，播放100万
用户: 分析一下为什么火
助手: 原因是XXX、YYY、ZZZ
用户: 好的，给我3个选题建议
助手: 建议A、B、C...
```

**压缩**：
```markdown
**对话摘要:**

**任务:** B站数据分析

**数据:**
- 粉丝: 5万
- 热门视频: XXX (100万播放)

**分析:** 火爆原因: XXX, YYY, ZZZ

**结果:** 3个选题建议 (A, B, C)
```

**节省**: 60% tokens

---

## ⚠️ 注意事项

1. **保留关键信息**：不要压缩掉重要细节
2. **保持逻辑完整**：压缩后逻辑要清晰
3. **可恢复性**：重要对话保存到MEMORY.md
4. **用户确认**：重要压缩需用户确认
