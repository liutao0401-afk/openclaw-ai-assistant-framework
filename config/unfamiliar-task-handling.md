# 🌐 陌生任务处理规则

## 🎯 核心原则

**不要闭门造车！遇到陌生任务，主动去学习！**

---

## 📋 识别标准

**什么是陌生任务？**

| 标准 | 说明 |
|:---|:---|
| **从未做过** | 第一次遇到的任务类型 |
| **复杂困难** | 超出当前能力范围的任务 |
| **缺少知识** | 缺乏必要的背景知识或技能 |
| **高风险** | 失败后果严重的任务 |

---

## 🔄 处理流程

### 第一步：识别陌生任务

**判断依据**：
1. 任务复杂度评估
2. 所需知识检查
3. 风险程度分析

---

### 第二步：主动学习

**学习来源**：

| 来源 | 说明 | 优先级 |
|:---|:---|:---:|
| **ClawHub** | 技能库，现成的skill或工具 | ⭐⭐⭐⭐⭐ |
| **GitHub** | 开源项目，代码示例 | ⭐⭐⭐⭐ |
| **B站/YouTube** | 视频教程，字幕提取 | ⭐⭐⭐ |
| **官方文档** | API文档，使用指南 | ⭐⭐⭐ |
| **博客/论坛** | 经验分享，问题解答 | ⭐⭐ |

---

### 第三步：组合应用

**策略**：
1. **直接使用**：找到现成的skill或工具
2. **学习后应用**：学习相关知识再执行
3. **组合创新**：组合多个工具解决问题

---

## 📚 学习方法

### ClawHub技能搜索

```bash
clawhub search "keyword"
clawhub install skill-name
```

**示例**：
- 任务：处理Excel文件
- 搜索：`clawhub search excel`
- 安装：`clawhub install xlsx`

---

### GitHub项目查找

```bash
gh search repos "keyword"
gh repo clone owner/repo
```

**示例**：
- 任务：视频处理
- 搜索：`gh search repos video-processing`
- 克隆：`gh repo clone user/video-tools`

---

### B站/YouTube字幕提取

**方法**：
1. 搜索相关视频
2. 提取字幕内容
3. 学习关键知识点
4. 应用到任务中

---

## 🎯 应用策略

### 策略1：直接使用现有工具

**适用场景**：有现成的skill或工具

**流程**：
1. 搜索ClawHub
2. 找到匹配技能
3. 直接安装使用

**示例**：
```
任务: 生成PDF报告
搜索: clawhub search pdf
安装: clawhub install pdf-skill
使用: 直接调用技能
```

---

### 策略2：学习后创建技能

**适用场景**：没有现成工具，但有相关知识

**流程**：
1. 搜索学习资料
2. 理解核心原理
3. 创建自定义skill
4. 应用到任务

**示例**：
```
任务: 自定义B站分析
学习: 查阅B站API文档
创建: 编写bilibili-analyzer skill
应用: 分析B站数据
```

---

### 策略3：组合多个工具

**适用场景**：单一工具无法完成

**流程**：
1. 分解任务步骤
2. 找到每个步骤的工具
3. 组合工具链
4. 执行完整流程

**示例**：
```
任务: 生成视频字幕
步骤1: 提取音频 → audio-extract skill
步骤2: 语音识别 → speech-to-text skill
步骤3: 生成字幕 → subtitle-generator skill
组合: 串联3个技能完成
```

---

## ⚠️ 注意事项

### 学习边界

**不要无限学习**：
- 学习时间 < 任务预计时间的30%
- Token消耗 < 总Token的20%
- 找到关键知识即可，不求完美

---

### 学习优先级

| 优先级 | 来源 | 时间分配 |
|:---|:---|:---:|
| **P1** | ClawHub技能 | 50% |
| **P2** | GitHub项目 | 30% |
| **P3** | 视频教程 | 15% |
| **P4** | 其他来源 | 5% |
